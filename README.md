Super Store Sales Dashboard (Power BI)

📌 1. Title & Brief Introduction

This project showcases an interactive Super Store Sales Dashboard built using Power BI to analyze business performance across regions, categories, and time.
It provides key insights into sales, profit, quantity, and delivery trends, helping stakeholders make data-driven decisions.

🎯 2. Short Description / Purpose

The main purpose of this dashboard is to:

Monitor overall sales performance
Analyze profit trends across different regions and categories
Identify top-performing segments and sub-categories
Track monthly sales growth (Year-over-Year)
Understand customer purchasing behavior and shipping preferences

This project simulates a real-world retail analytics scenario.

📊 4. Data Source

The dataset used in this project is based on a retail superstore dataset (commonly used for analytics practice).

📁 Data Structure Includes:
Order Details: Order ID, Order Date, Ship Date
Customer Details: Segment (Consumer, Corporate, Home Office)
Geography: Region (Central, East, South, West), State
Product Info: Category, Sub-category (Phones, Chairs, Binders, etc.)
Sales Metrics: Sales, Profit, Quantity
Shipping Info: Ship Mode (Standard, First Class, etc.)
Payment Mode: COD, Cards, Online

The dataset is structured in a tabular format, making it suitable for Power BI data modeling.

💼 6. Example Business Problem

Problem:
A retail company wants to improve its profitability and identify which areas are underperforming.

How this dashboard helps:

Identifies low-profit regions or states
Highlights high-selling but low-profit products
Tracks seasonal sales trends for better planning
Shows preferred shipping & payment methods
Helps optimize inventory and marketing strategies

Example Insight:

If a sub-category like Binders has high sales but lower profit, pricing or cost strategies may need adjustment.
If one region dominates sales, expansion strategies can focus on weaker regions.
